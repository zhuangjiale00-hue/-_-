作品簡介

「一體式智能盆栽管家」是一個以Arduino微控制器為核心的智能種植系統。它通過土壤濕度、溫度和光照傳感器實時監測植物生長環境，並依據科學設定的閾值，自動啟動水泵進行精準澆水。

更重要的是，系統將複雜的數據轉化為直觀的「健康值」、「口渴度」等遊戲化指標，通過LCD屏幕或簡單的網頁界面反饋給用戶，實現了從"環境感知"到"自動執行"再到"人性化交互"的完整閉環。

創新性

首創"監測-執行-遊戲化指引"三位一體的設計理念，專門服務於種植新手。

科學性

基於傳感器物理原理和閉環反饋控制理論，決策依賴實時客觀的環境數據。

實用性

低成本、易用性強，切實解決"養不活植物"的痛點，適用於多種場景。

生活觀察與設計原意

生活觀察

「植物殺手」的困境

在快節奏的都市生活中，許多人（尤其是學生、上班族）雖喜愛植物，卻因忙碌或缺乏經驗，常常忘記澆水、或不了解植物習性而導致植物枯萎。

科技與生活的脫節

現有的智能園藝設備要么過於複雜昂貴，要么功能單一（如僅有定時澆水），無法理解植物的真實需求，也無法與用戶進行有效溝通。

自然體驗的缺失

我們希望透過一個簡單的科技產品，重新建立人與自然的連結，讓種植變得輕鬆、有趣且有成就感。

設計原意

我們的初衷不是創造一個冷冰冰的機器，而是一位貼心的「植物管家」。它能代替我們：

  • 「看見」植物的乾渴
  • 「感受」環境的變化
  • 在需要時「行動」起來
  • 用我們聽得懂的語言「匯報」植物的狀態

我們希望通過最普及的計算機科學知識，解決普遍的生活痛點，讓科技成為承載關懷、培養責任感的橋樑。

詳細設計與科學原理

系統架構

傳感器層 - 土壤濕度、溫度、光照傳感器
控制決策層 - Arduino主板及程序
執行器層 - 繼電器、水泵
人機交互層 - LCD屏幕/網頁界面

硬件設計與科學性

感知模塊

土壤濕度傳感器：利用土壤的電導率原理，通過探針測量電阻值，間接獲得土壤的體積含水量。

溫濕度傳感器(DHT11)：採用熱敏電阻和濕敏電容測量環境溫濕度。

控制模塊

Arduino Uno：作為系統的"大腦"，負責運行預設的程序邏輯，處理所有傳感器數據並下達控制指令。

執行模塊

繼電器模塊：利用電磁效應，實現用Arduino輸出的低電壓信號控制水泵所需的高電壓電路。

微型水泵：根據指令完成灌溉動作。

軟件邏輯與創新性

核心控制算法 (防過度灌溉機制)：

// 偽代碼示例 - 防過度灌溉機制 void loop() { int moisture = readSoilMoisture(); // 讀取土壤濕度 if (moisture < THRESHOLD_DRY) { // 如果太乾 startWatering(); // 開始澆水 delay(WATERING_TIME); // 澆水持續一段時間 stopWatering(); // 停止澆水 delay(RE_CHECK_DELAY); // 【創新點】關鍵延遲！等待水分滲透 // 再次檢測土壤濕度 moisture = readSoilMoisture(); // 若依然乾燥，可能需要進一步處理 if (moisture < THRESHOLD_DRY) { // 可以觸發警報或採取其他措施 triggerAlert(); } } // 檢查其他環境因素 checkTemperature(); checkLightLevel(); // 更新顯示數據 updateDisplay(); }

此「澆水後延時再檢測」邏輯，模擬了人類"澆一點水，看看情況"的經驗，是一種簡單而有效的自適應性控制方法。

遊戲化交互設計：

程序內部定義了如 health_score 和 thirst_level 等變量。這些變量是通過對傳感器原始數據進行加權計算得出，並通過 map() 函數將傳感器讀數映射到更直觀的百分比或等級。

首創「監測-執行-遊戲化指引」深度融合的設計理念，專門服務於種植新手：

  • 不僅實現自動化，更注重用戶體驗

通過「澆水後延時再檢測」的軟件邏輯，在極簡硬件架構上實現了防過度灌溉的自適應控制機制：

  • 模擬人類澆水經驗
  • 防止系統振盪和資源浪費
  • 提高系統可靠性和科學性

3. 低成本教育實踐方案

構建了以"傳感器-單片機-繼電器"為核心的低成本硬件模型，突出其教育實踐價值：

  • 所有硬件均為常見開源模塊
  • 接線和編程相對簡單，易於複製
  • 適合作為高中科創教育案例

科學性分析

本作品基於嚴謹的科學原理和工程方法：

1. 傳感器物理原理應用

  • 土壤濕度傳感器：基於土壤電導率與含水量之間的關係
  • 溫濕度傳感器：基於熱敏電阻和濕敏電容的物理特性
  • 光照傳感器：基於光敏電阻的光電效應

2. 控制理論應用

  • 閉環反饋控制：系統根據傳感器反饋調整執行器狀態
  • 閾值控制：為不同植物設定科學的水分閾值
  • 自適應機制：通過延時再檢測防止過度灌溉

3. 數據驅動的決策

所有決策均基於實時、客觀的環境數據，而非預設的時間表：

  • 澆水決策基於土壤濕度實時測量值
  • 植物健康評估基於多傳感器數據融合
  • 系統行為可根據環境變化自適應調整
